## What this project is

`gitstats.py` — a single-file Python script that analyzes a git repository and produces
a self-contained, dark-themed HTML report with Chart.js charts and interactive tables.

**Output**: one `index.html` file, no external dependencies at runtime.

**Usage**: `python3 gitstats.py <repo_path> [output_dir]`

## Features implemented

- Commit count matching GitHub (`git rev-list HEAD --count`, not `--all`)
- Per-author commit list modal: click author name → modal with hash, date, message, GitHub link
- GitHub remote URL auto-detected (SSH + HTTPS)
- Lazy section init: charts only render when their nav tab is first clicked
- Chart.js charts: activity (hour/dow/month/year/monthly-line), heatmap, authors bar,
  contributions per-author, files-over-time, LOC-over-time
- Obfuscation + screenshot pipeline (`docs/screenshot.py`)

## Screenshot rendering — root cause + fix

**Problem**: Charts rendered inside `display:none` sections measure 0-width containers →
broken axis layouts (rotated labels eating vertical space, flat bars).

**Fix** (in `docs/screenshot.py`):
1. Patch HTML before loading: inject `<style>.section{display:block!important}</style>`
2. Call every `sectionInits[sec]()` after load — all charts render at correct 1400px width
3. Scroll full page so all content renders
4. Screenshot each section by PIL-compositing `header` + `nav` + `#sec-{name}` element
   screenshots — no show/hide toggling, no chart resize triggered

## Obfuscation rules (docs/screenshot.py)

- Author names: `AUTHOR_MAP` dict (real → fake)
- Short hashes `<code>` tags: deterministic MD5 fakes
- Full hashes in JSON: SHA1 fakes
- Commit messages: only fix unintelligible/revealing ones via `MESSAGE_FIXUPS`
- `githubBase` cleared to `""` — modal links don't point to real commits
- Repo path → `/private/Verwaltungssoftware`, remote URL → `[private repo]`
