## What this project is

`gitstats.py` — a single-file Python script that analyzes a git repository and produces
a self-contained, dark-themed HTML report with Chart.js charts and interactive tables.

**Output**: one `index.html` file, no external dependencies at runtime.

**Usage**: `python3 gitstats.py <repo_path> [output_dir]`

## Features implemented

- Commit count matching GitHub (uses `git rev-list HEAD --count`, not `--all`)
- Per-author commit list modal: click author name → modal with hash, date, message, GitHub link
- GitHub remote URL auto-detected from `git remote get-url origin` (SSH + HTTPS)
- Lazy section init: charts only render when their nav tab is first clicked
- Chart.js charts: activity (hour/dow/month/year/monthly-line), heatmap, authors bar,
  contributions per-author, files-over-time, LOC-over-time
- Obfuscation script (`docs/screenshot.py`) replaces author names + short hashes,
  fixes a handful of personally-revealing commit messages, clears GitHub base URL

## Screenshot rendering — root cause + fix

**Problem**: Charts rendered inside `display:none` sections measure 0-width containers.
Chart.js uses that 0-width to size axis labels, producing broken/squished layouts
(rotated labels eating all vertical space, flat bars, etc.). Contributions section was
unaffected because its charts are built dynamically on first click, after the section
is already visible.

**Failed approaches**:
- `add_style_tag` after `page.goto` — wiped by navigation
- `add_style_tag` after load, then `style.remove()` before screenshots — nav click
  triggers chart `resize()` which recalculates with wrong dimensions
- Destroy + reinit via `sectionInits[sec]()` after clicking nav — same resize problem

**Working fix** (in `docs/screenshot.py`):
1. Patch the HTML file before loading: inject `<style>.section{display:block!important}</style>` into `<head>`
2. After load, call every `sectionInits[sec]()` explicitly — all charts render at full 1400px width
3. Scroll full page so all content is rendered and animations complete
4. Per screenshot: use Playwright's **element screenshot** (`el.screenshot()`) on the
   `header`, `nav`, and `#sec-{name}` elements separately, then PIL-composite them
   into one image — no show/hide toggling, no chart resize triggered

## Obfuscation rules (docs/screenshot.py)

- Author names: replaced via `AUTHOR_MAP` dict (6 entries, real → fake Western names)
- Short hashes in `<code>` tags: replaced with deterministic MD5-based fakes
- Full hashes in JSON `authorCommits`: replaced with SHA1-based fakes
- Commit messages: only fix unintelligible/revealing ones via `MESSAGE_FIXUPS` dict
  (keep "reste migrations", LOC values, etc. as-is — they're fine)
- `githubBase` in JSON: cleared to `""` so modal links don't point to real commits
- Repo path: replaced with `/private/Verwaltungssoftware`
- Remote URL in subtitle: replaced with `[private repo]`
